# 用户须知 / USER GUIDE

## 1. 这是什么

Outlook 邮箱助手是一个**本地运行**的小工具，用来：

1. 导入你自己持有的 Outlook/Hotmail OAuth 凭证  
2. 批量检测账号是否仍可登录（验活）  
3. 读取收件箱/垃圾箱邮件并提取验证码或链接  
4. 给账号做备注、按状态/域名家族（Hotmail / Outlook / 其他）筛选、导出 Live/Dead  

它**不是**云端 SaaS，数据默认只存在你自己电脑上的 `app/data/mail.db`。

---

## 2. 适用 / 不适用

**适用**

- 管理你自己注册或已获授权的邮箱号池  
- 注册业务里「收验证码」自用工具  
- 本地批量检查 refresh_token 是否失效  

**不适用 / 不要用来**

- 未授权访问他人邮箱  
- 撞库、盗号、贩卖他人账号  
- 对微软服务进行攻击、滥用自动化造成干扰  

---

## 3. 安装与启动

### Windows

1. 解压发布包  
2. 双击 `Outlook邮箱助手.exe`  
3. 浏览器访问提示的地址（默认 `http://127.0.0.1:17890`）  
4. 首次登录：`admin` / `admin123`，**必须改密**  

### Linux

```bash
./outlook-mail-helper
```

需 PHP 8.1+（curl / openssl / mbstring / pdo_sqlite / sqlite3）。

关闭黑色终端窗口 = 停止服务。

## 4. 账号导入格式

每行一条：

```text
邮箱----密码----ClientID----RefreshToken
```

- 密码字段可填真实密码或占位  
- ClientID / RefreshToken 为微软 OAuth 凭证  
- 设置里可改分隔符  

支持：

- 文本粘贴批量导入  
- **TXT 文件导入（推荐）**：浏览器本地读取后**自动分块**提交，**无 80MB 总上限**（实测 1 万行约 5MB，10 万行约 50MB 均可直接导）  
- 小体积 JSON 数组导入（同邮箱会更新而非重复插入；**大 JSON 不推荐**，请转成 txt）  

说明：

- 旧版「超过 80MB 请拆分」已去掉；大号池请选文件 → 等进度条跑完即可  
- 同邮箱再次导入 = 更新 token，不重复插行  
- 单次粘贴仍受浏览器/请求体限制；超大请用「选择文件」

## 5. 常用操作

| 操作 | 说明 |
|------|------|
| 批量验活 | 后端独立进程并发检测；可关面板后台跑 |
| 全选当前 | 选中「当前筛选/搜索结果」全部卡片 |
| 删除选中 | 只删勾选的账号 |
| 删除全部 Live | 清空状态为 live 的账号 |
| 删除全部账号 | 清空数据库（二次确认） |
| 设置/追加备注 | 对选中账号写备注 |
| 导出 Live/Dead | 导出对应状态凭证文本 |
| 代理 | 设置 → 代理：HTTP 或 SOCKS5 |

---

## 6. 代理

路径：**设置 → 代理**

- 类型：HTTP / SOCKS5  
- 主机/端口：例如 `127.0.0.1` + `10809`  
- 点「测试代理连通」看出口 IP 与微软是否可达  

OAuth 刷 token、IMAP 探活、收件都会走代理（启用后）。

## 7. 数据与备份

| 路径 | 内容 |
|------|------|
| `app/data/mail.db` | 账号、设置 |
| `app/data/jobs/` | 验活任务状态与日志 |

备份：复制整个 `app/data` 目录即可。  
**mail.db 含 refresh_token，等同邮箱钥匙，勿明文外传。**

---

## 8. 安全建议

1. 仅绑定本机回环地址使用  
2. 立刻修改默认管理员密码  
3. 不要把安装目录同步到公开网盘  
4. 不要把服务端口映射到公网  
5. 用完可导出需要的 Live 列表后清理本地库

## 9. 常见问题

**Q: 验活很慢？**  
A: 看代理质量与是否开启 IMAP 探测。可在设置里调大并发（建议 10–30），或先关 IMAP 只刷 Token。

**Q: 关闭进度窗口任务停了吗？**  
A: 默认不会停。「后台运行/关闭面板」只关 UI；「停止任务」才会取消。

**Q: 为什么删不了？**  
A: 先勾选卡片，或用「删除全部 Live/全部账号」。清理 Dead 只删 dead 状态。

**Q: Linux 提示没有 php？**  
A: `sudo apt install php-cli php-curl php-sqlite3 php-mbstring php-xml`（包名因发行版而异）。
